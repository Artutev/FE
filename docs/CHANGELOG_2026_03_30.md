## 1️⃣ Темная тема

### ✨ Новые файлы

#### `static/js/theme.js` (новый файл)
JavaScript логика для управления темой:
- **Автоматический выбор** темы на основе системных настроек (`prefers-color-scheme`)
- **Сохранение выбора** в `localStorage` браузера

**Ключевые функции:**
```javascript
window.themeToggle.init()       // Инициализация темы при загрузке
window.themeToggle.toggle()     // Переключение темы
window.themeToggle.updateIcon() // Обновление иконки кнопки
```

### 🚀 Как использовать

1. Нажми кнопку 🌙 в заголовке
2. Тема переключается с плавным переходом

- Темные стили автоматически применяются при добавлении класса `dark-mode` на `<html>`

```css

.my-component {
    background: var(--surface);
    color: var(--text);
}

/* Неправильно */
.my-component {
    background: #ffffff;
    color: #1f2937;
}
```

## 2️⃣ Поиск мероприятий

### 📝 Измененные файлы

#### `Events/views.py`
**Обновлено:**
- Импорт `Q` из `django.db.models` для сложных запросов
- Функция `event_list()` теперь поддерживает поиск:
  ```python
  search_query = request.GET.get('search', '')
  if search_query:
      events = events.filter(
          Q(name__icontains=search_query) |
          Q(description__icontains=search_query) |
          Q(location__icontains=search_query)
      )
  ```
- Передача `search_query` в контекст шаблона

**Особенности:**
- Поиск **без учёта регистра** (`icontains`)
- **Поиск по нескольким полям** одновременно (название, описание, место)
- **OR логика** — результат содержит хотя бы одно совпадение

#### `Events/templates/eventList.html`
**Добавлено:**
- **Форма поиска** с полем ввода и кнопками:
  ```html
  <form method="get" class="search-form">
      <div class="search-container">
          <input type="text" name="search" class="search-input" 
                 placeholder="Поиск мероприятий..." value="{{ search_query }}">
          <button type="submit" class="search-btn">🔍</button>
          {% if search_query %}
              <a href="{% url 'Event:event_list' %}" class="search-clear">✕</a>
          {% endif %}
      </div>
  </form>
  ```

- **Информация о результатах** при поиске
- **Специальное сообщение** если ничего не найдено с предложением очистить поиск
- **Уникальные пустые состояния** для поиска и обычного отсутствия событий

#### `static/css/event_list.css`
**Добавлено:**
- **Стили поиска** с поддержкой тёмной темы:
  - `.search-form` — контейнер формы
  - `.search-container` — flex контейнер для инпутов
  - `.search-input` — поле ввода (48px высота)
  - `.search-btn` — кнопка поиска с иконкой 🔍
  - `.search-clear` — кнопка очистки поиска
  - `.search-results-info` — блок с информацией о результатах

- **Все стили адаптивны** и работают в темном режиме

### 🎯 Как работает поиск

1. **Введи текст** в поле поиска (минимум 1 символ)
2. **Нажми Enter** или кнопку 🔍
3. Страница перезагружается с параметром `?search=текст`
4. **Результаты фильтруются** по:
   - Названию события
   - Описанию события
   - Месту проведения

5. **Показывается счётчик** найденных мероприятий
6. **Кнопка ✕** очищает поиск и возвращает все события

### 📊 Примеры запросов

```
?search=футбол    → Мероприятия с "футбол" в названии, описании или месте
?search=москва    → События в Москве или содержащие "москва"
?search=лекция    → Лекции и события с этим словом
```

### ✅ Особенности

- **Нечувствителен к регистру** — "Футбол" = "ФУТБОЛ" = "футбол"
- **Работает с частичным совпадением** — поиск "фут" найдёт "футбол"
- **Поиск по нескольким полям** одновременно
- **Интегрирован с существующей сортировкой** по дате и времени
- **Поддержка тёмной темы** встроена в стили
